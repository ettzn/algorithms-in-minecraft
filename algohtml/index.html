💎 SEARCHING
ALGORITHM 01

Interpolation Search

Estimates where the target lives — like guessing a page in a phone book.

⚙ Analysis
Type
Searching
Space
\(O(1)\)
Avg Time
\(O(\log \log n)\)
Worst Time
\(O(n)\)
\[ pos = low + \frac{(target - arr[low])(high - low)}{arr[high] - arr[low]} \] Unlike Binary Search, it estimates position from value distribution. Best on uniformly distributed data.
🌍 Real-World Example
📖 Finding a word in a dictionary
PROBLEM
1,000-page dictionary. Find "butterfly." You skip to page ~50 because 'B' is early — not page 500.
SOLUTION
The algorithm interpolates a likely page from value distribution. Recalculates on sub-ranges. Much faster than binary search for uniform data.

ALGORITHM 02

DFS — Depth-First Search

Go deep down one path before backtracking. Stack-powered exploration.

⚙ Analysis
Type
Graph / Tree
Structure
Stack (LIFO)
Time
\(O(V+E)\)
Space
\(O(V)\)
Explores as far as possible down one branch before backtracking. Great for: puzzles, topological sort, cycle detection, connected components.
🌍 Real-World Example
🌀 Solving a maze
PROBLEM
Complex labyrinth, many dead-ends. No map. Must find exit without revisiting corridors.
SOLUTION
Pick a direction, go as deep as possible. Mark visited spots. Hit a dead-end? Backtrack. Recursion handles this naturally — finds a path, not necessarily shortest.

ALGORITHM 03

BFS — Breadth-First Search

Expand outward in layers. Guaranteed shortest path in unweighted graphs.

⚙ Analysis
Type
Graph / Tree
Structure
Queue (FIFO)
Time
\(O(V+E)\)
Space
\(O(V)\)
Explores ALL neighbors at current depth before going deeper. Best for: shortest path (unweighted graphs), web crawling, GPS navigation.
🌍 Real-World Example
🚇 Shortest subway route
PROBLEM
One Station → Another Station. Fewest stops (not shortest distance). Which route?
SOLUTION
BFS from Starting Station: visit all 1-stop stations first, then 2-stop, etc. The moment you hit Destination Station, that's the minimum — BFS guarantees it by expanding in layers.
🔴 SORTING
ALGORITHM 04

Merge Sort

Divide, conquer, and merge. Stable sort with guaranteed O(n log n).

⚙ Analysis
✅ STABLE — preserves order of equal elements
Type
Sorting
Space
\(O(n)\)
Time (all cases)
\(O(n \log n)\)
Splits array into halves recursively → merges sorted halves. Merging checks left element first on ties — preserving original order.
🌍 Real-World Example
🏆 Leaderboard: score + timestamp
PROBLEM
Players have (score, timestamp). Sort by score, but ties must keep earliest-first order.
SOLUTION
Stable sort preserves timestamp order from the first pass. Merge Sort checks left first on equal scores — ties never scramble. Quicksort (unstable) would break this.
🟣 SORTING
ALGORITHM 05

Quicksort

Pivot, partition, conquer. Fastest in practice thanks to cache locality.

⚙ Analysis
Type
Sorting
Space
\(O(\log n)\)
Avg Time
\(O(n \log n)\)
Worst Time
\(O(n^2)\) rare
Pick a pivot. Partition: <pivot left, >pivot right. Recurse on sub-arrays. In-place. Extremely fast due to CPU cache locality.
🌍 Real-World Example
📚 Library bookshelf by Dewey Decimal
PROBLEM
10,000 random books. Sort by call number quickly with minimal physical moves.
SOLUTION
Pick a pivot book (e.g., #500). Swap books <500 left, >500 right. Repeat on each half. In-place = minimum moves. Far faster than bubble or insertion sort.

ALGORITHM 06

Heapsort

Max-heap powered. Guaranteed O(n log n) in ALL cases — no exceptions.

⚙ Analysis
⚠ NOT STABLE · But worst-case guaranteed
Type
Sorting
Space
\(O(1)\) in-place
Time (all cases)
\(O(n \log n)\)
Build a max-heap (\(O(n)\) heapify). Repeatedly extract max to array end. Unlike Quicksort — worst case is still \(O(n \log n)\).
🌍 Real-World Example
🏥 Hospital ER triage queue
PROBLEM
Patients arrive with priority 1–5 (1=critical). Doctors must always treat the highest priority next. New patients keep arriving.
SOLUTION
Heapify the initial patient list in \(O(n)\). Max (highest priority) is always at root. New arrivals insert in \(O(\log n)\). This IS a live priority queue — Heapsort demonstrates its structure.
